
 🌐 OSI-Modell (Open Systems Interconnection)

##  Bedeutung
Das OSI-Modell beschreibt die Kommunikation zwischen Systemen in 7 Schichten.

Es hilft, Netzwerke besser zu verstehen und Fehler zu analysieren.


##  Die 7 Schichten

# 7 Anwendungsschicht (Application)
- Schnittstelle für Anwendungen
- Beispiel: HTTP, FTP


# 6️ Darstellungsschicht (Presentation)
- Datenformatierung (z. B. Verschlüsselung)


# 5️ Sitzungsschicht (Session)
- Verbindungsaufbau und -steuerung


# 4️ Transportschicht (Transport)
- Datenübertragung
- Beispiele: TCP, UDP



# 3️ Netzwerkschicht (Network)
- Routing von Datenpaketen
- Beispiel: IP


# 2️ Sicherungsschicht (Data Link)
- MAC-Adresse
- Fehlererkennung



# 1️ Bitübertragungsschicht (Physical)
- Übertragung von Bits (Kabel, Signale)


# Merksatz
"Alle Deutschen Studenten Trinken Verschiedene Sorten Bier"

(Application, Presentation, Session, Transport, Network, Data Link, Physical)


# Beispiele (wichtig!)

- HTTP → Schicht 7  
- TCP/UDP → Schicht 4  
- IP → Schicht 3  
- MAC-Adresse → Schicht 2  


##  Prüfungstipps

 Häufige Fragen:
- „Welche Schicht ist TCP?“ → Schicht 4  
- „Wo gehört IP hin?“ → Schicht 3  
- „Welche Schicht hat MAC-Adresse?“ →  Schicht 2  

 Wichtig:
- Reihenfolge der Schichten kennen  
- Beispiele zuordnen können  


##  Warum wichtig?
- Grundlage für Netzwerktechnik  
- Fehleranalyse im Netzwerk  
- Sehr häufig in Prüfungen gefragt  


#  Fazit
Das OSI-Modell ist ein fundamentales Konzept der Netzwerktechnik und hilft beim Verständnis der Datenkommunikation.
